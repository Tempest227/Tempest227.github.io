How to track and display profile views on GitHub
发表于 2020-07-24| 更新于 2020-08-11|计算机编程
阅读量:

原文链接

随着Github的部分网页设计发生改变，Github为个人简介增加了READMEs。通过创建一个带有你的名字的仓库并且用markdown编写一个README.md文件，你就可以丰富你的自我介绍。

不像以前只能用纯文本，新特性支持外部链接和图片。这就是有趣的地方。

追踪个人信息页

因为Github的markdown可以将外部的图片展示出来，你现在可以追踪并且展示个人信息。

想要这样做，你需要在服务器存一张图片然后通过Markdown语法将图片显示出来：

在服务端，每张图片的请求可以看作个人信息的展示。剩下的你只要在数据库存一个计数器，然后返回一张带有编号SVG图片作为响应。当然，虽然会有来自机器人的请求，但这样做好过什么都没有。

为了防止IP和User-Agent的渗透，GitHub通过特定的代理缓存所有的图片。

为了防止Github服务器在第一次图片请求就开始缓存，你的图片必须要指定缓存报文头或者返回non-200（比如：HTTP 404页面）。

这对于仓库来说不是什么新鲜的东西因为它存在有好长时间了。GitHub允许这样使用图片来做interactive badges。不像个人信息，仓库有权限追踪这些数据。

有很多库和服务可以为你做这些事。如果你想要有更多的功能，也可以使用Pipedream 。

安全方面

对很多人来说，在他们的网站允许外部的图片听着不是个事。我已经在不同平台使用这项技术超过10年。我经常可以看到，当招聘人员做了个决定，我的主页就变了。

不同于GitHub，大多数网站甚至不用代理去隐藏IP、referrer和browser agent。如果你想要在你的网站使用外部图片，你必须代理它们并隐藏请求它的人的所有信息。

一个怀有恶意的人可以欺骗受害者打开你看起来完全合法的资料，并探测到他的IP和浏览器。你也可以设置一个cookie来跟踪用户在其他网站上的访问，但是浏览器很快就会停止这样做。

单像素图像可以提供很多信息。这就是整个广告行业是如何跟踪你在不同网站上的访问。

结论

根据我的测试，个人信息视图不算是repo views，我希望在未来我们能够观察仓库的追踪状态。
